Makefile Documentation

1. help

Displays a list of available tasks with their descriptions.

make help

2. install-all

Installs all components required for the Kubernetes cluster, including kind (Kubernetes IN Docker), Nginx Ingress, MetalLB, and demo workloads.

make install-all

3. install-all-no-demo-workloads

Installs all components required for the Kubernetes cluster, excluding the demo workloads.

make install-all-no-demo-workloads

4. create-cluster

Creates a Kubernetes cluster using kind.

make create-cluster

5. export-cert

Exports Kubernetes keys (client) and certificates (client, cluster CA).

make export-cert

6. k8s-dashboard

Installs the Kubernetes dashboard.

make k8s-dashboard

7. nginx-ingress

Installs the Nginx Ingress controller.

make nginx-ingress

8. metallb

Installs the MetalLB load balancer.

make metallb

9. deploy-app-nginx-ingress-localhost

Deploys an httpd web server and creates an ingress rule for localhost (http://demo.localdev.me:80/). It also patches the ingress-nginx-controller service type to LoadBalancer.

make deploy-app-nginx-ingress-localhost

10. deploy-app-helloweb

Deploys the "helloweb" app.

make deploy-app-helloweb

11. deploy-app-golang-hello-world-web

Deploys the "golang-hello-world-web" app.

make deploy-app-golang-hello-world-web

12. deploy-app-foo-bar-service

Deploys the "foo-bar-service" app.

make deploy-app-foo-bar-service

13. ingress-rule

Adds an ingress rule to route traffic to different subdomains.

make ingress-rule

14. cluster-health-check

Checks the health status of the Kubernetes cluster and reports the objects and resources.

make cluster-health-check

15. generate-a-load-of-randomized-traffic

Generates random traffic to test the cluster's performance.

make generate-a-load-of-randomized-traffic

16. build-kubectl-test-image

Builds a Docker image for kubectl testing.

make build-kubectl-test-image

17. delete-cluster

Deletes the Kubernetes cluster.

make delete-cluster

18. kill-switch

DANGER ZONE! This is a kill switch that indiscriminately deletes all services and pods across all namespaces. Use with extreme caution.

make kill-switch

Note: Before using any of these commands, ensure you have the necessary permissions and understand the implications of executing certain actions, especially those marked as "DANGER ZONE" like the kill-switch. Always exercise caution and verify the impact of each command before executing it.
